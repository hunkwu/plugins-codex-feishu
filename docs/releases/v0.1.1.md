# Feishu for Codex v0.1.1

Maintenance release focused on installability, smoke testing, and Feishu event subscription support.

## Added

- `scripts/smoke-test.sh` for local plugin structure and MCP protocol checks.
- `plugins/feishu/scripts/feishu_webhook_server.py` as a minimal Feishu event callback receiver.
- `plugins/feishu/scripts/test-feishu-webhook.py` for local webhook fixture checks.
- `plugins/feishu/skills/feishu/examples/webhook-to-reply.md` for the event-to-reply workflow.
- Webhook reference documentation with Feishu Open Platform configuration checklist.
- `CONTRIBUTING.md` with PR process and contribution scope.
- `case-studies/` with a bilingual case study template for community submissions.
- Private assistant push documentation explaining `App ID` vs recipient `open_id`.
- `scripts/check-sensitive-values.sh` to catch real-looking Feishu IDs or secrets before release.

## Webhook Capabilities

- Handles `url_verification` by returning the incoming `challenge`.
- Verifies `FEISHU_VERIFICATION_TOKEN` when configured.
- Verifies `X-Lark-Signature` when `FEISHU_ENCRYPT_KEY` is configured.
- Decrypts encrypted event bodies when `FEISHU_ENCRYPT_KEY` is configured.
- Logs normalized events to stdout or `FEISHU_WEBHOOK_EVENT_LOG`.

## Verification

Run:

```bash
scripts/smoke-test.sh
```

This validates plugin JSON, required files, executable scripts, MCP `initialize`, MCP `tools/list`, and webhook self-test behavior.

The smoke test also validates plaintext webhook fixtures, `im.message.receive_v1`, signature calculation, and encrypted payload decryption.

## 中文说明

`Feishu for Codex v0.1.1` 聚焦安装稳定性、本地冒烟测试和飞书事件订阅 Webhook 支持。

### 新增

- `scripts/smoke-test.sh`：本地插件结构与 MCP 协议检查。
- `plugins/feishu/scripts/feishu_webhook_server.py`：轻量飞书事件回调接收服务。
- `plugins/feishu/scripts/test-feishu-webhook.py`：本地 Webhook fixture 检查。
- `plugins/feishu/skills/feishu/examples/webhook-to-reply.md`：事件到回复 workflow 示例。
- Webhook 参考文档：包含飞书开放平台配置清单。
- `CONTRIBUTING.md`：贡献范围与 PR 流程说明。
- `case-studies/`：双语实战案例模板，面向社区投稿。
- 私人助理推送文档：说明 `App ID` 与接收人 `open_id` 的区别。
- `scripts/check-sensitive-values.sh`：发布前扫描真实飞书 ID 或密钥。

### Webhook 能力

- 处理 `url_verification`，返回飞书传入的 `challenge`。
- 配置后校验 `FEISHU_VERIFICATION_TOKEN`。
- 配置 `FEISHU_ENCRYPT_KEY` 后校验 `X-Lark-Signature`。
- 配置后解密 `FEISHU_ENCRYPT_KEY` 加密事件体。
- 将标准化事件输出到 stdout 或 `FEISHU_WEBHOOK_EVENT_LOG`。

### 验证方式


该脚本会验证插件 JSON、必要文件、脚本可执行权限、MCP `initialize`、MCP `tools/list` 和 Webhook 自测。

同时会验证明文 Webhook fixtures、`im.message.receive_v1`、签名计算和加密事件解密。
