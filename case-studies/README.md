# Case Studies

Real stories from builders using Codex, Feishu, MCP, and agent workflows to ship useful projects.

[中文说明](#中文说明)

## What Belongs Here

This directory collects practical case studies, especially stories about:

- Shipping a product or internal tool with Codex.
- Using Feishu + Codex to coordinate work, summarize messages, search knowledge, or push project updates.
- Improving an `AGENTS.md` file until Codex became more reliable in a real repo.
- Launching quickly, validating demand, getting first users, or earning first revenue.
- Debugging failures that other builders can learn from.

## How To Submit

1. Copy [`TEMPLATE.md`](./TEMPLATE.md).
2. Create a new file using this naming style:

```text
case-studies/YYYY-MM-DD-short-project-name.md
```

3. Remove private data before submitting.
4. Keep the story concrete and useful.
5. Open a PR and link any related issue, demo, repo, or public launch post.

## Editorial Bar

Useful case studies usually include:

- Starting point and constraints.
- The exact workflow or rules that helped.
- What shipped.
- Time spent.
- Result or learning.
- What you would change next time.

The goal is not to make every story look successful. Honest postmortems are valuable.

## 中文说明

# 实战案例库

这里收集开发者使用 Codex、飞书、MCP 和智能体 workflow 推动真实项目落地的案例。

## 收录什么

优先收录这些故事：

- 用 Codex 上线产品或内部工具。
- 用 Feishu + Codex 做协作、群消息总结、知识库检索或项目进展推送。
- 持续调优 `AGENTS.md`，让 Codex 在真实 repo 中更稳定。
- 快速上线、验证需求、获得第一批用户或赚到第一笔收入。
- 记录失败排查过程，让其他开发者少踩坑。

## 如何投稿

1. 复制 [`TEMPLATE.md`](./TEMPLATE.md)。
2. 按这个命名方式新建文件：

```text
case-studies/YYYY-MM-DD-short-project-name.md
```

3. 提交前移除私密信息。
4. 内容保持具体、有用。
5. 发起 PR，并附上相关 issue、demo、repo 或公开发布链接。

## 内容标准

有价值的案例通常包括：

- 起点和约束。
- 真正起作用的 workflow 或规则。
- 实际上线了什么。
- 花了多久。
- 结果或经验。
- 下次会怎么改。

案例库不只收成功故事。真实复盘和失败经验同样有价值。
